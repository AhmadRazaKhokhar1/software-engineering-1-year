# UI/UX Design (with Framer) — 4-Month Syllabus

This track assumes **8–10 hours per week**. Each month is roughly **four weeks**. Framer is used for **high-fidelity prototyping** and **interactive presentation**; Figma (or equivalent) may be paired for broader team workflows where noted.

---

## Month 1 — UX Foundations & User-Centered Process

- UX vs UI; usability, utility, and desirability
- Design thinking: discover → define → ideate → prototype → test
- User research basics: interviews, surveys, competitive analysis
- Personas, scenarios, and journey maps
- Information architecture: navigation patterns, content grouping
- Accessibility mindset: contrast, focus, touch targets, motion sensitivity (intro)
- **Project:** Research brief + low-fidelity wireframes for one mobile-first flow (6–8 screens)

---

## Month 2 — Visual Interface Design & Systems

- Layout, spacing scales, and grid fundamentals
- Typography hierarchy and pairing; readability on small screens
- Color systems: semantic tokens (brand, surface, text, danger)
- Iconography and imagery: style, cropping, consistency
- Components and patterns: buttons, forms, lists, empty states, errors
- Handoff readiness: specs, naming, and annotation habits
- **Project:** UI style tile + component set applied to the Month 1 flow (mid-fidelity)

---

## Month 3 — Framer: Prototyping & Interaction Design

- Framer workspace: pages, frames, responsive breakpoints
- Building a clickable prototype: transitions, overlays, scrolling regions
- Micro-interactions: hover/press (where applicable), timed sequences (intro)
- Using components/variants for reusable UI (as supported by current Framer workflows)
- Embedding video, maps, or embeds for realistic marketing sites (where appropriate)
- Prototype testing: tasks, success criteria, and note-taking
- **Project:** Framer prototype of a marketing landing + 2 key user flows with interactive states

---

## Month 4 — UX Validation, Iteration & Portfolio Delivery

- Usability testing: moderated sessions, think-aloud, severity ratings
- Iterating from feedback; prioritization (impact vs effort)
- UX metrics: task success, time-on-task, error rate, SUS (intro)
- Case study storytelling: problem, constraints, process, decisions, outcomes
- Collaboration with developers: edge cases, states, and documentation quality
- **Capstone:** Polished case study (PDF/slides + Framer link) covering research → IA → UI → prototype → test results → next steps

---

## Outcomes

Graduates should be able to **frame UX problems**, **design coherent interfaces**, **prototype credibly in Framer**, and **communicate decisions** in a portfolio-ready case study.
